Time to Completion
  • Null Hypothesis: the time to completion for version A is equal to that for version B.
  • Alternative Hypothesis: the time to completion for version B is shorter than that for version A, because version B emphasize the checkout button, which indicates the action much more clearly.
Return Rate
  • Null Hypothesis: the return rate for version A is equal to that for version B.
  • Alternative Hypothesis: the return rate for version B is larger than that for version A, becuase version B does not only emphasize the price, so people are more likely to return to see if they reach $150.

Part 2: Collecting Data and Applying Scripts

Collecting Data

After I finished writing the HTML and upload it to Heroku, I collect all data through the studio. There are 13 users who have valid data using version A, and another 13 users using version B. There is also one user who has valid data using both version A and version B. Then I used the filtered log document to calculate time to completion and return rate for each version.

Takeaways
  • Larger sample size will detect the significance of the difference more accurately, right now the sample size is not very large, which is less likely to reject the null hypothesis.
  • It's crucial to focus on the specific metrics we would like to test first, only after which we could get statistic differences using statistic methods.
  • A/B testing is a great way to tell the difference between two interfaces by recording users' behavior and calculate result quantitively.
  • Before calculating the result, it's very important to first clarify the null hypothesis and the alternative hypothesis, because it should based on every lttle difference between interfaces.
